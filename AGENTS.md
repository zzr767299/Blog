# AGENTS.md

## Overview

Next.js 15 App Router blog using Contentlayer2 for MDX content, Tailwind CSS v4, and React 19. Based on `tailwind-nextjs-starter-blog` v2.4.0.

## Commands

```bash
yarn install          # install deps (Yarn 3.6.1, node_modules linker)
yarn dev              # dev server at localhost:3000 (uses cross-env INIT_CWD)
yarn build            # production build + RSS postbuild step
yarn lint             # ESLint with --fix on app/components/lib/layouts/scripts dirs
yarn serve            # start production server
```

**Windows caveat**: On Windows, set `$env:PWD = $(Get-Location).Path` before running `yarn dev` or `yarn build` if `cross-env` causes issues. The scripts use `INIT_CWD=$PWD`.

**Static export**: `EXPORT=1 UNOPTIMIZED=1 yarn build` generates `out/` folder for static hosting. Add `BASE_PATH=/path` for subpath deploys.

## Pre-commit

Husky runs `lint-staged` on commit: ESLint `--fix` + Prettier on staged JS/TS/MDX/JSON/CSS files. No separate `typecheck` script exists; TypeScript checking happens via ESLint.

## Content Architecture

- **Blog posts**: `data/blog/**/*.mdx` — must have `title` and `date` frontmatter
- **Authors**: `data/authors/**/*.mdx` — must have `name` frontmatter; `default.md` is required
- **Site config**: `data/siteMetadata.js` (analytics, comments, search, newsletter)
- **Navigation**: `data/headerNavLinks.ts`
- **Projects page**: `data/projectsData.ts`
- **Generated artifacts**: `app/tag-data.json` (auto-generated by Contentlayer on build), `public/search.json` (kbar index)

## Contentlayer Quirks

- Uses `contentlayer2` (not the original unmaintained `contentlayer` package)
- Content source directory is `data/` (set in `contentlayer.config.ts` `contentDirPath`)
- Post slug is derived by stripping the first path segment from `flattenedPath` (e.g. `blog/foo` → `foo`)
- Nested routes work: `data/blog/nested-route/post.mdx` is valid
- `onSuccess` callback auto-generates `app/tag-data.json` and optionally `public/search.json`
- If Contentlayer cache breaks, delete `.contentlayer/` and rebuild

## Path Aliases

`@/components/*`, `@/data/*`, `@/layouts/*`, `@/css/*` — defined in both `tsconfig.json` and `jsconfig.json`. Contentlayer generated types are at `contentlayer/generated`.

## Key Config Files

- `next.config.js` — CSP headers, SVG loader, Contentlayer + Bundle Analyzer plugins
- `contentlayer.config.ts` — MDX remark/rehype plugins, content schema
- `eslint.config.mjs` — flat config format, ignores `next-env.d.ts` and `next.config.js`
- `prettier.config.js` — no semicolons, single quotes, 100 width, tailwindcss plugin
- `postcss.config.js` — uses `@tailwindcss/postcss` (Tailwind v4 style)

## Frontmatter Fields

```yaml
title: string (required)
date: date (required)
tags: string[] (optional)
lastmod: date (optional)
draft: boolean (optional)
summary: string (optional)
images: json (optional)
authors: string[] (optional, defaults to ['default'])
layout: string (optional: PostLayout | PostSimple | PostBanner)
canonicalUrl: string (optional)
```

## Deployment

GitHub Pages via `.github/workflows/pages.yml` — runs static export (`EXPORT=1 UNOPTIMIZED=1`) and deploys `out/` folder. Triggered on push to `main`.

## Styling

- Tailwind CSS v4 with `@tailwindcss/postcss` plugin
- Prism code highlighting via `css/prism.css`
- Custom MDX components registered in `components/MDXComponents.tsx`
